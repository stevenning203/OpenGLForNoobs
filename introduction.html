What is OpenGL?

OpenGL is a rendering API which allows us to communicate with graphics units in order to produce high performance computer graphics. It is not a library, and it is not as simple as #include ing a single header. We will need multiple libraries in order to do what we want with OpenGL. Here they are:

There are a bunch of headers. First let's start with the most crucial: glad and glfw.

glad

glad is a header that gets function pointers for OpenGL functions. Since there are many OpenGL versions that have many different functions, we have to retrieve them somehow. This is done through glad. glad loads all the functions that we will use in order to draw what we want. However, all glad does is load the functions we need to actually DRAW the shapes and colour them. To create our window context and have input methods, we use glfw.

Graphics Library Framework (glfw)

glfw allows us to create our window that we will render to and process input like mouse input and keyboard input. This includes things such as mouse position, keyboard presses, and mouse clicks. It is essential to us being able to create the window and process input.

Strictly speaking, the next headers are not necessary, but they make our lives much, much easier.

GL Mathematics (glm)

glm is a header that includes many math tools we can use to help create what we want. It includes matrices, vectors, and functions that allow us to manipulate matrices and vectors. Theoretically, you could create this header yourself with relative ease, but it's there, so we can just use it.

Open Asset Import Library (assimp)

assimp is a library that makes model loading a lot easier. It processes textures and vertices that will be used when we want to draw models. Although it is technically a model loader, we still have to do a lot of the work ourselves because we have to convert it into a format we can use to draw. assimp is a great help to loading assets.

stb_image

stb_image is a single header that is used to load images. We use it alongside assimp to load raw image data in the form of bytes to create the textures that are mapped onto the models loaded with assimp.

That's all for the non-standard headers!
